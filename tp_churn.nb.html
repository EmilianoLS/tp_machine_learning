Handling Outliers

Comnmente se considera Outlier a todo registro que est fuera de lo comn, todo registro que resalte del resto. Esto no es algo muy definido, ya que no existe una nica regla para determinar qu es lo comn.

Por eso existen diversas formas para detectar estos valores extremos que no parecieran seguir el mismo comportamiento que el resto, nosotros nicamente uno en particular:

  • Mtodo de IQR o del Rango intercuartilico

Mtodo IQR

Conceptualmente, se determina que un valor es un outlier para una determinada variable, cuando el mismo se encuentra por encima o por debajo de ciertos lmites, determinados por el rango intercuartlico o IQR. Estos lmites son:

\(\left[(Q1 - 1.5 IQR) - (Q3 + 1.5 IQR)\right]\)

Por eso, calculamos los rangos intercuartlicos y los Q1 y Q3 para cada variable numrica del dataset, y calculamos estos lmites para cada una:

# Calculo el primer cuantil de cada variable y lo guardo en un df
Q1 <- as.data.frame(apply(select_if(data, is.numeric), 2, quantile, probs = 0.25))

# Cambio los nombres y agrego una variable
Q1$variable <- rownames(Q1)
rownames(Q1) <- c(1:nrow(Q1))
names(Q1)[1] <- 'quantile_25'

# Calculo el tercer cuartil de cada variable y lo guardo en un df
Q3 <- as.data.frame(apply(select_if(data, is.numeric), 2, quantile, probs = 0.75))
Q3$variable <- rownames(Q3)
rownames(Q3) <- c(1:nrow(Q3))
names(Q3)[1] <- 'quantile_75'

# Unifico ambos df
quantiles_df <- left_join(Q1,Q3, by = 'variable')
# Calculo el rango intercuartilico
quantiles_df$IQR <- quantiles_df$quantile_75 - quantiles_df$quantile_25

El principal objetivo es reducir la dimencionalidad del dataframe. Para esto combinaremos una serie de modelos que determinen la importancia de los atributos, y utilizaremos la combinacin de esos resultados para determinar cules son las variables ms importantes.

Antes de eso, sin embargo, haremos un poco de feature engineering sobre las variables. Primeramente, haremos tres pruebas:

  • Una transformacin logartmica sobre el conjunto de datos
  • Una normalizacin sobre el conjunto de datos
  • La combinacin de las dos anteriores

Todas las pruebas se evaluarn sobre el conjunto de validacin luego de entrenarse los modelos, para verificar si alguna de ellas mejora la performance sobre el modelo benchmark. El motivo para tales transformaciones, especialmente la logartmica, es porque muchas de las variables se encuentran muy concentradas en valores pequeos mientras que los valores ms grandes tienen pocos registros. Esto significa que las variables no siguen una distribucin normal, y una transfomacin logartmica puede ayudar a normalizar la distribucin.
